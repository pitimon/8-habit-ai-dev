# Script vs AI Workflow — the Execution/Verification Boundary

**Status**: Canonical reference (v2.21.36) | **Consumers**: skill authors, contributors deciding "should this be a script or markdown instructions?" | **Friction citation**: user portability request 2026-07-03 ([#354](https://github.com/pitimon/8-habit-ai-dev/issues/354)) — "ปรับ script เป็น AI workflow เพื่อใช้ได้โดยไม่จำกัดว่าต้อง shell execute"

**Habit**: H1 (Be Proactive — degrade gracefully before the constrained host appears) + H8 (Spirit — a self-graded check is a conscience violation, not a convenience).

## The principle

> **An AI workflow may replace a script's _execution_ when a deterministic verifier gates the result downstream. It must never replace a script's _verification_.**

Two reasons, both already paid for in this repo's history:

1. **Verification must be deterministic.** The validators are fitness functions — same input, same verdict, exit code CI can trust. An LLM re-deriving 40+ structural checks is probabilistic self-grading: the exact grade-saturation failure the retired SELF-CHECK table demonstrated (frozen 5.0 self-scores vs the honest 3.6 external audit — see `SELF-CHECK.md`, Fable F4) and the "false success report" anti-pattern the plugin teaches.
2. **Execution is recoverable.** If the model performs a mechanical step wrong (a missed file copy, a malformed report), the downstream deterministic gate catches it — `sync-mirror.sh` done by hand with Read/Write still has to pass Check 28's `diff -qr`. Wrong execution fails loudly; wrong verification fails silently.

Corollary for generated artifacts: regenerate through the generator, never hand-edit its output — the `--check` drift gates treat hand edits as staleness.

## What this means for portability

- **End users installing the plugin need no shell for the core value.** Skills, habits, guides, and rules are pure markdown; the only user-side executables are the session hook (which has a markdown degradation path) and the `/ai-dev-log` generator (which now has a documented AI-workflow fallback).
- **Contributors need bash + node** (or CI) — that is a maintainer-surface requirement, and it is deliberate: it keeps verification deterministic.

## When authoring a new skill

Default to markdown instructions (the existing Bash-minimization rule in `CLAUDE.md` §Proposed). Reach for a script only when the work is (a) mechanical and volume-heavy, or (b) a verification gate. If (a), document an AI-workflow fallback in the skill body. If (b), it belongs in `tests/` or `hooks/` and must never gain an LLM substitute.
