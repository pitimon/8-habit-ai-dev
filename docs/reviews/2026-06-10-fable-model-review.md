### Credit where due (verified positives)

- Zero false-success patterns in CI proper: all `|| true` matches in test scripts are deliberate `grep -c` zero-match guards; all 4 suites propagate failure (`set -euo pipefail` + explicit exit 1). All GitHub Actions SHA-pinned; lychee runs with `fail: true`.
- Dual-mirror sync is _real_ today: Check 28 `diff -qr` + `cmp -s` enforcement verified, 3 sampled mirrored files byte-identical.
- When-to-Skip sections are honest, not token gestures — `/save-spec` argues against its own use when memory-MCP is active.
- v2.21.20 PATH hardening verified correct: `/bin/bash` absolute, system dirs prepended with `${PATH:-}` guard, Codex fallback for `CLAUDE_PLUGIN_ROOT`.
- No secrets, no eval/source of user-controlled content in hooks; Codex JSON path properly escapes. Profile-injection surface exists (`session-start.sh:98` echoes raw `level:` value) but requires home-dir write access — low risk.
- Claims spot-check: skills-count badge, gbrain citation, and "≤300 tokens, enforced" all backed by evidence.

## Shadow Self-Check (adversarial pass on this verdict)

Strongest counter-argument: most findings are documentation drift on a single-maintainer repo whose release-integrity machinery is demonstrably green — severity could be inflated, and fixing them costs maintainer time better spent on cited friction. Response: F1 is not drift — it is shipped breakage affecting every installer of `/diagnose`, invisible precisely because the validator suite cannot see it; and F4 (grade saturation) means the repo's primary honesty instrument would never have caught any of this. **fix-then-ship** stands. Who is harmed if this verdict is wrong: the maintainer over-invests in polish (bounded, recoverable); if the counter-argument wins and F1 ships unfixed, every adopter of the flagship friction-driven skill silently loses its reference content (unbounded, invisible).
